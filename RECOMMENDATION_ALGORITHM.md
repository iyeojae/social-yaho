# Recommendation Algorithm Overview

## Goal
이 추천 알고리즘의 목표는 다음 두 가지를 동시에 만족하는 것입니다.

1. **회원가입 직후에도 어느 정도 개인화된 추천을 제공**한다.
2. **사용자가 실제로 읽고 반응하는 데이터가 쌓일수록 추천 결과가 계속 보정**되도록 한다.

즉, 이 시스템은 **초기 설문 기반 개인화 + 행동 데이터 기반 지속 학습** 구조입니다.

---

## 1. 전체 구조

추천 로직은 크게 3단계로 작동합니다.

### 1) 회원가입 설문 기반 초기 선호도 설정
회원가입 시 사용자는 아래 3가지 온보딩 설문 정보를 줄 수 있습니다.

1. 직접 선택한 선호 장르
2. **What kind of story pulls at your heart the most?**
3. **Tell us about a recent book, film, or drama that stayed with you.**

이 서술형 답변은 GPT(OpenAI)를 활용해 기존 장르 코드 집합으로 추론됩니다.
즉, 설문 결과는 최종적으로 다시 **장르 점수 형태**로 환원됩니다.

예:
- `CLASSIC`
- `POETRY`
- `FOLKTALE`

이 점수는 사용자가 아무 책도 읽지 않았을 때 **cold start 문제를 줄이는 역할**을 합니다.

### 2) 읽기/좋아요/북마크 행동 기반 선호도 누적
사용자가 서비스를 이용하면서 남기는 상호작용 이벤트가 장르 선호도에 계속 반영됩니다.

예:
- VIEW
- READ_START
- READ_COMPLETE
- LIKE
- BOOKMARK
- UNLIKE
- UNBOOKMARK

이 데이터는 `implicitScore`로 누적됩니다.

### 3) 초기 설문 + 행동 데이터 합산으로 추천
최종 추천은 아래 두 점수를 합쳐서 계산합니다.

- `explicitScore`: 회원가입 설문 기반 초기 취향
- `implicitScore`: 실제 읽기/반응 기반 동적 취향

최종 선호도 점수:

```text
totalScore = explicitScore + implicitScore
```

추천 엔진은 이 `totalScore`를 기준으로 상위 선호 장르를 뽑고, 최근 읽은 책 데이터와 함께 추천 후보를 정렬합니다.

---

## 2. 저장되는 선호도 데이터

사용자-장르 선호도는 `user_genre_preferences` 테이블에 저장됩니다.

각 레코드는 다음 정보를 가집니다.

- `user_id`
- `genre_id`
- `explicit_score`
- `implicit_score`
- `total_score`

### explicit_score
회원가입 설문에서 사용자가 직접 선택한 장르에 부여되는 점수입니다.

현재 정책:

```text
직접 선택한 장르 1개당 explicitScore +10
서술형 답변으로 추론된 장르 1개당 explicitScore +6
```

이렇게 분리한 이유는 다음과 같습니다.

- 직접 선택한 장르는 사용자의 명시적 의사를 가장 강하게 반영
- 서술형 답변 기반 추론 장르는 심리적/정서적 성향을 보조적으로 반영

즉, 사용자가 직접 고른 장르가 우선이고,
GPT로 추론한 장르는 **초기 개인화의 보강 신호**로 사용됩니다.

### implicit_score
사용자의 실제 행동으로부터 누적되는 점수입니다.

현재 정책:

- `VIEW` -> +1
- `READ_START` -> +2
- `READ_COMPLETE` -> +5
- `LIKE` -> +4
- `BOOKMARK` -> +3
- `UNLIKE` -> -2
- `UNBOOKMARK` -> -1

이 점수는 책 자체가 아니라, **그 책이 가진 장르들**에 누적됩니다.

예를 들어 사용자가 `CLASSIC`, `NOVEL` 장르가 붙은 책에 `LIKE`를 하면:

- `CLASSIC` implicitScore +4
- `NOVEL` implicitScore +4

---

## 3. 회원가입 설문이 추천에 주는 영향

회원가입 직후에는 읽기 이력이 없기 때문에 행동 기반 추천이 거의 불가능합니다.
이때 설문이 초기 추천을 시작하게 해줍니다.

예:
- 사용자가 가입 시 `CLASSIC`, `POETRY`를 선택
- 그리고 서술형 답변에서 "lyrical", "grief", "memory", "quiet emotional change" 같은 성향이 드러남
- GPT 또는 fallback 규칙이 `POETRY`, `CLASSIC`, `ESSAY` 같은 장르를 추가 추론
- 아직 읽은 책이 0권이어도
- 해당 장르 도서가 먼저 추천 후보로 올라갑니다.

이 덕분에 서비스 첫 진입부터 “취향과 무관한 랜덤 추천”이 아니라,
**초기 관심 장르 중심 추천**이 가능합니다.

---

## 4. 시간이 지나면 취향이 바뀌는 문제

사용자는 처음 설문에서 고른 취향과 실제 읽는 취향이 달라질 수 있습니다.
이 문제를 해결하기 위해 설문 점수와 행동 점수를 분리했습니다.

### 예시
처음 가입 시:
- `POETRY` 선택 -> explicitScore +10

이후 실제 행동:
- `CLASSIC` 도서 여러 권 읽음
- `CLASSIC` LIKE / READ_COMPLETE 반복

그 결과:
- `POETRY`는 초기에 강했지만
- `CLASSIC`의 implicitScore가 계속 쌓이면서
- 최종 `totalScore`는 `CLASSIC`이 더 높아질 수 있습니다.

즉,
**설문은 시작점이고, 실제 행동 데이터가 시간이 지나면서 추천 방향을 계속 수정합니다.**

---

## 5. 추천 후보 선정 방식

추천 서비스는 다음 순서로 작동합니다.

### Step 1. 상위 선호 장르 조회
`user_genre_preferences`에서 `totalScore` 기준 상위 5개 장르를 가져옵니다.

### Step 2. 최근 읽은 책 조회
최근 상호작용한 서로 다른 책 최대 5권을 가져옵니다.

### Step 3. 최근 읽은 책 장르 추출
최근 읽은 책들의 장르를 모읍니다.

### Step 4. 추천 타겟 장르 집합 생성
아래 두 집합을 합칩니다.

- 설문 + 행동 기반 상위 선호 장르
- 최근 읽은 책 장르

### Step 5. 후보 책 조회
위 장르에 속하면서, 최근 읽은 책은 제외한 활성 도서를 가져옵니다.

### Step 6. 후보 책 점수 계산
각 후보 책은 아래 요소를 합산해 점수를 받습니다.

- 선호 장르와 일치하는 개수
- 최근 읽은 장르와 일치하는 개수
- 장르별 totalScore
- 장르별 explicitScore
- 장르별 implicitScore

현재 계산 개념은 다음과 같습니다.

```text
score =
  (matchedPreferredCount * 30)
+ (matchedRecentCount * 25)
+ min(totalPreferenceScore, 30)
+ min(explicitScore, 20)
+ min(implicitScore, 25)
```

이 구조의 의미는 다음과 같습니다.

- 설문 기반 초기 관심도도 반영한다.
- 하지만 실제 읽기 데이터도 강하게 반영한다.
- 둘 중 하나만 극단적으로 커져도 점수가 과도하게 폭주하지 않도록 상한을 둔다.

---

## 6. 추천 이유 생성 방식

추천 피드와 추천 이유 API는 단순히 책만 내보내는 것이 아니라,
왜 추천됐는지를 함께 설명할 수 있습니다.

추천 이유는 아래 요소를 조합합니다.

- 최근 읽은 책
- 추천 대상 책과 겹치는 장르
- 누적 선호 장르
- 사전 생성된 AI 추천 문구(`BookAiTag.recommendationReason`)

예시:

```text
최근 읽으신 '운수 좋은 날'과 비슷한 관심 흐름이 보여 'Hong Gil Dong'을 추천드려요.
특히 고전문학 장르 취향과 잘 맞습니다.
```

이 구조 덕분에 프론트는 단순 추천이 아니라,
**설명 가능한 추천(explainable recommendation)** UI를 만들 수 있습니다.

---

## 7. 메타데이터 도서(XLSX)와 원문 도서(MD)의 차이

### MD 기반 도서
- 원문/설명 데이터가 풍부함
- 상세조회 시 원래 설명을 그대로 사용

### XLSX 기반 도서
- 메타데이터만 있고 설명이 빈약할 수 있음
- 상세조회 시 자동으로 AI 요약을 보강 가능
- 외국인 대상 사이트 기준으로 AI 요약은 **영어로 생성**됨

즉 추천 알고리즘과 별개로,
**메타데이터 도서의 상세 UX도 AI로 보강**되는 구조입니다.

---

## 8. 왜 이 구조가 적합한가

이 구조는 해커톤/초기 서비스에 적합합니다.

### 장점
1. **Cold Start 대응 가능**
   - 설문이 초기 추천 품질을 높임

2. **사용자 행동 기반 적응 가능**
   - 읽기 이력, 좋아요, 북마크가 계속 반영됨

3. **설명 가능한 추천 가능**
   - 추천 이유 API 제공 가능

4. **구현 복잡도가 과하지 않음**
   - JPA + Redis 캐시 + 점수 누적 방식으로 충분히 운영 가능

5. **확장 가능**
   - 나중에 작가 선호도, 언어 선호도, 국가 선호도, 번역 언어 선호도 등으로 확장 가능

---

## 9. 현재 운영 정책 요약

### 회원가입 설문
- 장르 최대 5개 직접 선택
- 선택 장르마다 explicitScore +10
- 서술형 답변 2개 입력 가능
- 서술형 답변은 GPT로 장르 코드 추론
- 추론 장르마다 explicitScore +6

### 행동 점수
- VIEW +1
- READ_START +2
- READ_COMPLETE +5
- LIKE +4
- BOOKMARK +3
- UNLIKE -2
- UNBOOKMARK -1

### 추천 계산
- 상위 선호 장르 + 최근 읽은 장르 조합
- 최근 읽은 책은 추천 대상에서 제외
- explicit + implicit를 함께 반영

---

## 10. 앞으로 확장 가능한 방향

다음 단계에서 확장할 수 있는 아이디어는 아래와 같습니다.

1. **설문 문항 고도화**
   - 서술형 답변 외에 감정 키워드/주제 키워드 선택형 추가

2. **행동 데이터 세분화**
   - 읽은 시간, 완독률, 재방문, 스크롤 깊이 등 반영

3. **부정 피드백 강화**
   - 숨김, 관심 없음, 빠른 이탈 등 반영

4. **작가/국가/번역 언어 선호도 추가**
   - 장르 외 특징까지 개인화 가능

5. **배치 재학습**
   - 하루 1회 사용자의 누적 패턴을 재정렬

---

## Final Summary

이 추천 시스템은 다음 철학으로 구성되어 있습니다.

- **처음에는 설문으로 시작하고**
- **이후에는 실제 행동으로 계속 수정되며**
- **시간이 지날수록 더 정교해지는 개인화 추천 시스템**

즉,

> "초기 취향은 직접 선택 장르 + 심리형 서술 답변으로 잡고,
> 실제 취향은 읽기 데이터를 통해 계속 학습한다."

이것이 현재 추천 알고리즘의 핵심입니다.
